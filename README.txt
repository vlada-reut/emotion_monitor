Emotion Monitor
===============

Назначение
----------
Программа предназначена для эмоционального мониторинга аудитории: получает видеопоток с камеры, обнаруживает лица, определяет эмоции, пол и возрастную группу, показывает аналитику в интерфейсе и сохраняет результаты локально.

Структура проекта
-----------------
Основные папки и файлы:

analytics/      - расчет статистики, распределений и итоговых сводок.
configs/        - настройки приложения. Основной файл: configs/settings.json.
core/           - обработка видеопотока, детекция лиц, сопровождение лиц, рабочий поток.
services/       - конфигурация, база данных, анализ лица, погода, журналирование.
ui/             - графический интерфейс PySide6.
weights/        - файлы моделей. Обязательный файл: weights/face_yolov8n.pt.
!docs/          - проектная документация: ТЗ, РСП и пояснительная записка.
main.py         - точка входа в приложение.
diagnose.py     - проверка зависимостей, настроек, модели и доступности папок.
requirements.txt - список Python-библиотек.
setup_env.bat   - первичная настройка на новом компьютере.
run_app.bat     - запуск приложения двойным кликом.

Папки data/ и logs/
-------------------
Эти папки не обязательно переносить на новый компьютер.

data/ создается автоматически при запуске приложения и хранит локальную SQLite-базу пользователей.
logs/ создается автоматически при запуске мониторинга и хранит журнал работы, JSONL-наблюдения и итоговые JSON-отчеты сессий.

Переносить data/ и logs/ нужно только в том случае, если требуется сохранить старую историю пользователей и прошлые отчеты. Для чистой установки их переносить не нужно.

Порядок запуска на новом компьютере
-----------------------------------
1. Установить Python 3.10 или совместимую версию Python 3.
2. Скопировать папку проекта emotion_monitor на новый компьютер.
3. Открыть папку проекта.
4. Один раз запустить setup_env.bat двойным кликом.
5. Дождаться установки библиотек и успешной проверки diagnose.py.
6. Для обычного запуска приложения использовать run_app.bat.

Файл setup_env.bat создает виртуальное окружение .venv, устанавливает зависимости из requirements.txt и запускает проверку проекта.

Файл run_app.bat запускает приложение без ручного открытия терминала.

Обычная работа с программой
---------------------------
1. Запустить run_app.bat.
2. В главном окне нажать кнопку "Начать мониторинг".
3. Держать лицо в зоне обзора камеры. Для накопления статистики и идентификации пользователя желательно находиться в кадре несколько секунд.
4. Для завершения сессии нажать "Остановить".
5. Закрыть окно приложения стандартными средствами Windows.

Где смотреть результаты
-----------------------
После запуска мониторинга программа автоматически создаст папку logs/.

В logs/ появляются:

app.log                         - общий журнал работы приложения.
session_YYYYMMDD_HHMMSS.jsonl   - подробные наблюдения по кадрам.
session_summary_YYYYMMDD_HHMMSS.json - итоговая сводка сессии.

База пользователей хранится в:

data/emotion_monitor.db

Проверка работоспособности
--------------------------
Если приложение не запускается или работает некорректно, сначала запустить diagnose.py:

1. Открыть папку проекта.
2. Запустить setup_env.bat, если окружение еще не создано.
3. При необходимости открыть терминал в папке проекта и выполнить:

.venv\Scripts\python.exe diagnose.py

В норме все пункты проверки должны иметь статус [ OK ].

Возможные проблемы
------------------
Если камера не открывается, нужно проверить, что она подключена и не используется другой программой.

Если камера имеет другой номер, изменить параметр camera_source в configs/settings.json. Обычно используется 0, иногда может потребоваться 1.

Если погодные данные недоступны, программа продолжит работу без них. Для получения погоды нужен доступ к интернету.

Если файл weights/face_yolov8n.pt отсутствует, мониторинг не запустится, потому что модель детекции лиц обязательна.
